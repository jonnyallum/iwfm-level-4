# Contributing

## Production Standard

This repository is intended to become a high-quality Facilities Management learning resource.

Do not add:

- placeholder modules
- empty folders
- unfinished lessons
- vague study notes
- easy questions pretending to be manager-level assessment
- uncited compliance claims where official guidance exists

## Required Module Checklist

Before a module is complete, confirm:

- [ ] Learning outcomes are clear.
- [ ] Lessons are phone-readable.
- [ ] Practical FM examples are included.
- [ ] Scenario questions are included.
- [ ] Hard questions are included.
- [ ] Model answers are included.
- [ ] Marking criteria are included.
- [ ] Interview talking points are included.
- [ ] Links to Compliance Bible are included where relevant.
- [ ] `TODO.md` is updated if a task is completed.
- [ ] `CHANGELOG.md` is updated.
- [ ] `python3 tools/check_repo.py` passes (links, metadata, module structure).

## Quality Test

A module is good enough only if the learner can answer:

1. What is the FM principle?
2. Why does it matter operationally?
3. What would I do in a real site?
4. What evidence would prove I handled it properly?
5. How would I explain it in an interview?
6. How would I apply it inside FM Control Hub or Compliance Hub?

## Difficulty Standard

Questions should test judgement, not memory.

Good question:

> A fire risk assessment identifies 45 non-compliances. Budget allows only 10 corrective actions this quarter. Explain your prioritisation method and evidence trail.

Weak question:

> What does FM stand for?
